This module analyzes customer purchasing behaviors on a weekly frequency basis. It loads cleaned transactional data and unique customer/product data, computes behavioral metrics, and outputs a processed dataset enriched with insights.
This module loads cleaned transaction data and unique customer records, enriches the transactions with time-based features, and then analyzes each customer’s behavior by calculating their average days between purchases as well as identifying their most frequent shopping day and hour. These behavioral insights are merged back into the customer dataset and saved as a new pickle file containing the final processed customer-behavior profile.
This module loads the processed customer-behavior data and defines a simple test that verifies whether key columns—specifically `CustomerID` and `Hour`—exist in the resulting DataFrame, raising an assertion error if the customers_behavior function did not produce the expected output structure.
This module loads a transaction dataset and filters out rows where the `StockCode` contains only zero or one numeric character—values considered anomalous since valid stock codes typically have five or six digits—before saving the cleaned DataFrame to a new pickle file and returning its path.
This module analyzes how order cancellations impact the business by identifying cancelled transactions, counting how often each customer cancels orders, and calculating their cancellation rate relative to total transactions. It merges these metrics into the existing customer dataset derived from geographic features and saves the enhanced DataFrame as a pickle file containing detailed cancellation behavior.
This module loads a cleaned transaction dataset, removes entries whose descriptions indicate service-related items such as next-day carriage or high-resolution image files, converts all remaining product descriptions to uppercase for consistency, and saves the refined DataFrame to a new pickle file before returning its path.
This module loads configuration settings and a parquet dataset, computes a correlation matrix, and visualizes it using a masked heatmap with a custom colormap before saving the plot and the full correlation matrix to specified output paths. It includes validation for data types, paths, and correlation thresholds, and provides helper functions to safely save both the heatmap and the parquet file, prompting the user when overwriting is necessary.
This function saves a heatmap figure as a PNG file by creating the target directory if it doesn't exist, attempting to write the image, and—if the file is already open or locked—prompting the user to decide whether to overwrite it before completing or aborting the save operation.
This function saves a correlation matrix as a Parquet file by first creating the target directory if it doesn't exist, then attempting to write the DataFrame; if the input is not a DataFrame, it raises an `AttributeError`, and if the file is already open or locked, it prompts the user to decide whether to overwrite it before completing or aborting the save.
